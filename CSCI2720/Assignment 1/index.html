Responsive Web Features

Those elements will be changed according to screen size.
Elements Changes
All other section except message box The format will be (8 colomns - 4 columns (nextline) 4 columns - 8 columns) in computer, but in mobile it will become 1 section full-width
Message Box When there is middle-size screen (e.g. tablet, fold phone), the message-box will occupy 6-colomns instead of either 4-colomns or full-width
Image When it is not large screen for computer, it will become smaller and only occupy 4 columns
Navbar When access it in mobile phone, the narbar links will become a botton at the right hand side

For bonus part

  • Dark mode: If your device is set as dark mode preferred, dark mode will be applied! The main theme color will become darker, including the section background color and the body background image. (This part used SASS to finish)
  • TypeScript: It declares the type of variable explicitly, there are many types of variable that TypeScript can declare, like HTMLElement, String, Integer, etc. However, after compile it into JavaScript, it will become the original JavaScript. TypeScript ensures the type of the variable is correct to prevent wrong type error when it is in JavaScript. (For example, anyElement.style.display can only be string but cannot be null in TypeScript. So if I want to remove the “none” display attribute, I need to use an empty string.
